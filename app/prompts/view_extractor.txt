以下のテキストから、各《ビュー》を説明する【文章】を抽出してください。
一つの文章が複数の《ビュー》に割当てても構いません。
ただし、必ず原文からの抜き出しのみとし、言い換えは禁止します。
曖昧・一般論・手がかり不足の文は抽出しないでください。
出力は必ず JSON のみ。

《ビュー》定義:
- value_analysis: プロジェクトのステークホルダを確認できる文章。また、「安心」「信頼」「利便性」「コスト削減」などステークホルダが受ける価値やペインの除去を説明する文章。
- business_concept: プロジェクトのビジョンや方針、コンセプトを説明する文章。理念的・方向性に関する記述。
- business_requirement: 業務遂行のために必要な機能やサービスを説明する文章。また、業務手続きや運用ルールを説明する文章。業務観点の要求を対象とする。
- system_requirement: システムが実現すべき具体的な技術仕様や非機能要件を説明する文章。実装条件や性能・セキュリティを含む。
- system_component: システムを構成するモジュール、ライブラリ、サービス、ハードウェアなどの要素を説明する文章。
- data_analysis: データの分析目的、分析手法、分析対象や結果を説明する文章。

出力は次の JSON オブジェクトだけ:
{
  "value_analysis": ["..."],
  "business_concept": ["..."],
  "business_requirement": ["..."],
  "system_requirement": ["..."],
  "system_component": ["..."],
  "data_analysis": ["..."]
}
